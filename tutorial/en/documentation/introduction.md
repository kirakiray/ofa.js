# Introduction

## What is ofa.js?

ofa.js is an easy-to-use front-end framework for building web applications, designed to boost development efficiency and lower the barrier to entry.

If you are already basically familiar with HTML, CSS, and JavaScript, then ofa.js is an excellent choice.

## Why am I developing ofa.js？

Although front-end frameworks like React and Vue exist, they come with a steep learning curve. Developers need to first acquire a series of prerequisite knowledge, such as installing Node.js, using npm to install packages, setting up scaffolding, and more.

These initial steps are not essential for actual development but instead become a "black box" in the project startup phase, burdening applications with unnecessary complexity from the very beginning. The birth of ofa.js is precisely to break this predicament.

## Purpose of ofa.js

### Streamline Front-End Development Workflow

ofa.js simplifies the cumbersome process of traditional front-end development. Developers can quickly build front-end applications without needing to learn tools like Node.js, npm, and scaffolding. Simply open the officially provided build program (web version) in a browser, select a local directory, and start developing.

### Lowering the Development Threshold

Many people dream of building their own app or becoming a developer, yet the traditional web (including Android, iOS, etc.) path demands so much up-front knowledge—installing dev environments, configuring boilerplates—that most beginners give up before they start.

ofa.js is dedicated to making everything simple. With just a Chrome browser, you can open a webpage and start building apps—select and open a local folder directly, skipping all pre-development setup. All computation, data, and storage run locally on the user’s machine, with no reliance on cloud services.

### The Ultimate Solution for Micro-Frontends

ofa.js enables Web frontend development to be subdivided into multiple independent modules, similar to microservices, where each module can be independently developed and deployed. Once the boundaries of traditional Web frontend are broken, frontend technology will gradually expand beyond the boundaries of server-side technology.

Microservices is a software architectural style that breaks down large, complex applications into multiple fine-grained, independently deployable, and runnable small services.
