# 介绍

## 什么是 ofa.js？

ofa.js 是一款易用的 Web 前端框架，专为构建 Web 应用而设计，旨在提高开发效率，降低开发门槛。

> 如果你对 HTML、CSS 和 JavaScript **已经基本熟悉**，那么 ofa.js 是一个非常好的选择。

## 我为什么要开发 ofa.js

ofa.js 的设计初衷很简单：在不依赖编译工具的情况下，解决 Web 开发的**工程化**问题。

> 工程化是指将软件开发过程中的规范、流程、工具和方法进行系统化整合，以提高开发效率、代码质量和可维护性。

近十年来，Web 前端开发经历了从初期野蛮生长到逐步工程化的演变过程。这一过程借鉴了传统应用开发的经验，通过引入 Node.js 和编译流程，解决了大型项目的工程化难题。

然而当项目变大后，这种模式的弊端逐渐暴露出来，出现了传统开发中面临的**巨石应用**问题，导致项目难以维护，交互需求迭代困难。

> 巨石应用（Monolithic Application）是指一个庞大且紧密耦合的单体应用，所有功能模块都集中在一个代码库中，牵一发而动全身，难以独立开发和部署。

这时候就需要对项目进行拆解，实现微前端化，就像**微服务**那样。但由于经过编译处理，微前端变得困难和繁琐，前端模块的独立部署需要对每个小模块进行编译，这是很困难的，导致 Web 前端技术的发展几乎面临停滞。

> 微服务（Microservices）是一种软件架构风格，它将大型、复杂的应用拆分为多个细粒度、独立部署和运行的小型服务。

这时候，我开始思考：传统编程语言需要应对不同的硬件和操作系统，所以必须通过编译来实现跨平台无差别化。但 Web 开发不一样，它基于浏览器，原本就无需编译、可以独立运行和部署，天生就是微前端模式。于是我意识到，是编译流程反而让事情变复杂了。

也就是说，只要解决工程化问题，然后去掉编译这个必要步骤，前端开发就很适合开发大型应用了，这就是天生的微前端模式。于是 ofa.js 应运而生。

## 核心优势

### 零门槛，开箱即用

无需搭建开发环境、安装依赖或配置脚手架，通过浏览器打开官方构建程序，选择本地目录即可开始开发。所有计算、数据和存储操作均在本地执行，无需依赖云端服务。

### AI 友好，易于验证

没有编译黑箱，AI 产出的代码可以快速部署和自验证；减少中间层，不走编译流程，让代码更容易定位问题和修复。

### 微前端原生支持

ofa.js 让 Web 前端开发可以像微服务一样，细分为多个独立模块，每个模块均可独立开发、独立部署。当传统 Web 前端的边界被打破后，前端技术将逐步突破服务器技术的限制。
