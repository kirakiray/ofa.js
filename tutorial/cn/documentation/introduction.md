# 介绍

## 什么是 ofa.js？

ofa.js 是一款易用的 Web 前端框架，专为构建 Web 应用而设计，旨在提高开发效率，降低开发门槛。

> 如果你对 HTML、CSS 和 JavaScript **已经基本熟悉**,那么 ofa.js 是一个非常好的选择。

## 为什么我要开发 ofa.js？

我认为前端架构中，本不应该存在 Node.js 之类的中间层进行"编译"。"编译"这个过程是借鉴传统应用开发经验而引入的。早期的前端开发不够系统化，甚至缺乏模块化支持，因此参考了大量成熟的应用编译流程，引入了 Node.js 等工具对网页应用进行"编译",以支持复杂应用的开发。

传统应用开发需要应对不同的硬件底层和操作系统，因此通过编译来实现跨平台无差别化，从而提高开发效率，避免繁琐的适配工作。

但 Web 前端天生就不需要面对这些问题。它的优势在于即刻运行——代码修改后能迅速看到效果，非常适合快速验证和迭代，这是 Web 与生俱来的优势。

然而，近十年的前端开发沿用了传统应用开发的经验，编译流程让事情变得异常复杂，使 Web 失去了这一核心优势，让应用开发重新回到了"编译"时代。而沿着编译这条路走下去，最终都难以避免"巨石应用"的困境。

传统的 Web 前端应用天生就是微前端架构，但为了解决模块化和工程化问题，却丧失了"无需编译"这一巨大优势。

我认为，Web 开发在解决模块化和工程化的同时，应该保留其快速开发的易用性（不依赖编译流程），这才是正确的发展方向。因此，我开发了 ofa.js。

## ofa.js 的优势

虽然已有 React、Vue 等前端框架，但它们都存在较高的上手门槛。开发者需要先学习一系列前置知识，例如安装 Node.js、使用 npm 安装包、搭建脚手架等。

这些初始步骤对实际开发并非不可或缺，反而成为了项目启动阶段的"黑箱",让应用从一开始就背负了不必要的复杂度。

在 AI 盛行的时代，ofa.js 因为没有黑箱，能让 AI 产出代码后快速进行部署和自验证。而且减少了中间层，不走编译流程，让代码更容易查找问题和修复。

## ofa.js 存在的目的

### 简化前端开发流程

ofa.js 简化了传统前端开发的繁琐流程。开发者无需学习 Node.js、npm 和脚手架等工具，即可快速搭建前端应用。只需通过浏览器打开官方提供的构建程序（网页版），然后选择本地目录，便可开始开发。

### 降低开发门槛

许多人希望开发属于自己的应用，或成为一名开发者，但面对传统 Web（包括安卓、iOS 等）开发路线时，却因需要学习大量前置知识，例如安装开发环境、配置脚手架，这让许多初学者知难而退。

ofa.js 致力于让一切变得简单。只需一个 Chrome 浏览器，打开网页即可构建应用，直接选择并打开本地目录，省去开发前的准备工作。所有计算、数据和存储操作均在用户本地执行，无需依赖云端服务。

### 成为微前端的终极解决方案

ofa.js 让 Web 前端开发可以像微服务一样，细分为多个独立模块，每个模块均可独立开发、独立部署。当传统 Web 前端的边界被打破后，前端技术将逐步突破服务器技术的边界。

> 微服务（Microservices）是一种软件架构风格，它将大型、复杂的应用拆分为多个细粒度、独立部署和运行的小型服务。
