# Utiliser les compétences IA d'ofa.js dans un projet

Étant donné que ofa.js n'est pas encore un framework largement connu, les modèles d'IA courants ne sont pas encore capables d'utiliser directement ofa.js. C'est pourquoi nous avons soigneusement préparé un pack de compétences dédié pour aider l'IA à apprendre et consulter la méthode d'utilisation d'ofa.js.

## Utilisation

Parmi les outils d'IA prenant en charge les compétences, sélectionnez l'adresse de compétence correspondante en fonction du modèle d'IA que vous utilisez :

**Attention :** En raison du fait que la documentation d'ofa.js a été initialement rédigée en chinois, le contenu du paquet de compétences en version chinoise est plus complet et précis. Si vous rencontrez des problèmes avec la version anglaise, il est recommandé d'essayer la version chinoise.

### Modèle chinois

Si vous utilisez un modèle d'IA chinois，veuillez utiliser l'adresse suivante：

```
https://raw.githubusercontent.com/ofajs/ofa.js/main/skills/ofajs-docs.zip
```

Les modèles d'IA chinois courants incluent : DeepSeek, Kimi, GLM, MiniMax, Qwen, etc.

### Modèle anglais

Si vous utilisez un modèle d'IA en anglais, veuillez utiliser l'adresse suivante :

```
https://raw.githubusercontent.com/ofajs/ofa.js/main/skills/ofajs-docs-en.zip
```

Grâce à ce pack de compétences, l'IA peut apprendre et développer efficacement des composants ou des modules de page pour ofa.js, aidant les développeurs à utiliser plus facilement les outils d'IA pour le développement de projets ofa.js et à améliorer l'efficacité du développement.

## Obtenir plus d'aide

Si les compétences en IA n'ont pas résolu votre problème, n'hésitez pas à poster votre question sur notre forum :

https://github.com/ofajs/discussion/discussions

Nous vous répondrons dans les plus brefs délais.
