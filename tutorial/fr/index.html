Je suis débutant, Node.js et npm m’ont rebuté

ofa.js est très accueillant pour les débutants : aucun Node.js, npm ou outil complexe requis.

Un simple navigateur et n’importe quel serveur statique suffisent pour démarrer.

Avec Chrome, ouvrez l’outil en ligne OFA Studio : plus besoin de serveur statique, vous codez directement.

Je développe un immense site e-commerce, pouvez-vous le supporter ?

ofa.js excelle dans les projets larges grâce à son architecture micro-frontend.

Vous pouvez découper la boutique en besoins fins, développer chaque composant dans son propre dossier et le prévisualiser via un simple serveur statique.

À la fin, vous assemblez le système complet. Le mode micro-frontend d’ofa.js est le remède idéal aux applications monolithiques.

Voir l’exemple NoneOS, un système d’exploitation de grande envergure développé avec ofa.js.

Je suis dev back-end et je veux seulement utiliser un moteur de template côté serveur ; ofa.js convient-il ?

Le SSR d’ofa.js n’est pas lié à Node.js : en respectant la règle SCSR, vous pouvez utiliser n’importe quel langage back-end pour vos templates.

Héritier spirituel de jQuery

Prend en charge la syntaxe des templates tout en permettant la manipulation directe des nœuds, combinant flexibilité et efficacité

La conception de l'API est similaire à celle de jQuery, mais plus intuitive

      
// Code jQuery
$("#target").html("some html code"); // Définir le html
$("#target").text("set text"); // Définir le texte
var ele_text = $("#target").text(); // Obtenir le texte
var parents = $("#target").parents(); // Obtenir un tableau de tous les parents
var child = $("#target").children()[0]; // Obtenir le premier élément enfant
      
    
      
// Code ofa.js
$("#target").html = "some html code";  // Définir le html
$("#target").text = "set text"; // Définir le texte
var ele_text = $("#target").text; // Obtenir le texte
var parents = $("#target").parents; // Obtenir un tableau de tous les parents
var child = $("#target")[0]; // Obtenir le premier élément enfant
