# Introducción

## ¿Qué es ofa.js?

ofa.js es un marco de trabajo de front-end web muy fácil de usar, diseñado para construir aplicaciones web, con el objetivo de mejorar la eficiencia del desarrollo y reducir el umbral de entrada.

>Si ya estás **bastante familiarizado** con HTML, CSS y JavaScript, entonces ofa.js es una excelente opción.

## ¿Por qué desarrollé ofa.js?

Aunque ya existen frameworks frontend como React y Vue, todos ellos presentan una barrera de entrada relativamente alta. Los desarrolladores deben aprender primero una serie de conocimientos previos, como instalar Node.js, utilizar npm para instalar paquetes y configurar un entorno de scaffolding.

Estos pasos iniciales no son indispensables para el desarrollo real, sino que se convierten en una "caja negra" durante la fase de inicio del proyecto, cargando la aplicación con una complejidad innecesaria desde el principio. El nacimiento de ofa.js tiene como objetivo precisamente romper este dilema.

## Propósito de ofa.js

### Simplificar el flujo de trabajo del desarrollo frontend

ofa.js simplifica el tedioso proceso del desarrollo frontend tradicional. Los desarrolladores no necesitan aprender Node.js, npm ni herramientas de andamiaje para crear rápidamente aplicaciones frontend. Solo tienen que abrir el programa de construcción oficial (versión web) en el navegador, seleccionar un directorio local y comenzar a desarrollar.

### Reducir el umbral de desarrollo

Muchas personas desean desarrollar sus propias aplicaciones o convertirse en desarrolladores, pero al enfrentarse a la ruta tradicional de desarrollo web (incluyendo Android, iOS, etc.), se desaniman al tener que aprender una gran cantidad de conocimientos previos, como instalar el entorno de desarrollo y configurar el andamiaje.

ofa.js se dedica a hacer que todo sea sencillo. Solo necesitas un navegador Chrome, abre una página web y podrás construir aplicaciones, selecciona y abre directamente un directorio local, ahorrando los preparativos previos al desarrollo. Todos los cálculos, operaciones de datos y almacenamiento se ejecutan localmente en el usuario, sin necesidad de depender de servicios en la nube.

### La solución definitiva para convertirse en micro-frontends

ofa.js permite que el desarrollo frontend web se subdivida, al igual que los microservicios, en varios módulos independientes, cada uno de los cuales puede ser desarrollado y desplegado por separado. Cuando los límites tradicionales del frontend web se rompen, la tecnología frontend irá traspasando gradualmente las fronteras de la tecnología del servidor.

> Los microservicios (Microservices) son un estilo de arquitectura de software que divide una aplicación grande y compleja en varios servicios pequeños, de granularidad fina, desplegados y ejecutados de forma independiente.
