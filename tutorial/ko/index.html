개발 초보인데, Nodejs랑 npm 때문에 포기했어요

ofa.js는 초보자에게 매우 친숙하며, Node.js, npm 등의 도구를 배울 필요가 없습니다.

브라우저 하나와 아무 정적 서버 하나만 있으면 개발을 시작할 수 있습니다.

Chrome 브라우저를 사용한다면, OFA Studio 이 온라인 도구를 열면 정적 서버를 준비할 필요 없이 바로 시작할 수 있습니다.

튜토리얼: 첫 번째 애플리케이션 만들기
거대한 쇼핑몰 시스템을 개발하는데, 여러분이 감당할 수 있나요?

ofa.js는 마이크로 프론트엔드 아키텍처 패턴을 채택하기 때문에 대규모 프로젝트에 매우 적합합니다.

쇼핑몰 시스템을 충분히 세분화된 요구사항으로 나누어, 각각의 컴포넌트를 개발하고 독립된 폴더에 배치한 후, 정적 서버를 통해 컴포넌트 효과를 바로 미리 볼 수 있습니다.

마지막으로 하나씩 조립하여 거대한 시스템을 만듭니다. ofa.js의 마이크로 프론트엔드 모드는 바로 모놀리식 애플리케이션을 해결하는 좋은 방법입니다.

사례 NoneOS를 참고할 수 있습니다. 이는 ofa.js로 개발된 운영체제로, 규모가 매우 방대합니다.

저는 백엔드 개발자인데, 그냥 백엔드 템플릿 엔진으로 개발하고 싶어요, ofa.js를 사용해도 괜찮을까요?

ofa.js의 SSR은 Node.js와 결합될 필요가 없습니다. SCSR 규칙만 준수한다면, 어떤 백엔드 언어로든 템플릿 엔진 개발을 할 수 있습니다.

jQuery의 정신적 계승자

템플릿 구문을 지원하면서도 직접 노드를 조작할 수 있어 유연성과 효율성을 모두 잡았습니다

API 설계는 jQuery와 유사하지만 더 직관적입니다

      
// jQuery 코드
$("#target").html("some html code"); // html 설정
$("#target").text("set text"); // text 설정
var ele_text = $("#target").text(); // text 가져오기
var parents = $("#target").parents(); // 모든 부모 계층 배열 가져오기
var child = $("#target").children()[0]; // 첫 번째 자식 요소 가져오기
      
    
      
// ofa.js 코드
$("#target").html = "some html code";  // html 설정
$("#target").text = "set text"; // text 설정
var ele_text = $("#target").text; // text 가져오기
var parents = $("#target").parents; // 모든 부모 계층 배열 가져오기
var child = $("#target")[0]; // 첫 번째 자식 요소 가져오기
