더 이상 배우기 싫어, 문서 읽기 싫은데 쓸 수 있어?

ofa.js를 먼저 배우지 않아도, 이를 기반으로 개발된 컴포넌트를 바로 재사용할 수 있습니다.

오픈소스 컴포넌트 작성자가 코드를 작성하면, 다른 사람들은 예제를 보고 몇 줄만 복사해 자신의 프로젝트에 기능을 즉시 가져올 수 있습니다.

레거시 프로젝트에 임시로 기능 하나 추가하고 싶어

기존 프로젝트에 script 태그로 ofa.js를 불러오기만 하면, 즉시 사용할 수 있습니다.

불러온 후, 다양한 컴포넌트를 바로 사용할 수 있습니다.

개발 초보인데 Nodejs랑 npm 때문에 포기했어

ofa.js는 초보자에게 매우 친숙하며, Node.js, npm 같은 도구를 배울 필요가 없습니다.

브라우저 하나와 아무 정적 서버 하나만 있으면 개발을 시작할 수 있습니다.

Chrome 브라우저를 사용한다면, OFA Studio 이 온라인 도구를 열면 정적 서버를 준비할 필요 없이 바로 시작할 수 있습니다.

튜토리얼: 첫 번째 애플리케이션 만들기
대규모 쇼핑몰 시스템 개발하는데, 감당 가능해?

ofa.js는 마이크로 프런트엔드 아키텍처 패턴을 채택하기 때문에 대규모 프로젝트에 매우 적합합니다.

쇼핑몰 시스템을 충분히 세분화된 요구사항으로 나누어, 각 컴포넌트를 개발하고 독립된 폴더에 배치하면, 정적 서버를 통해 컴포넌트 효과를 바로 미리 볼 수 있습니다.

마지막으로 하나씩 조립하여 거대한 시스템을 만듭니다. ofa.js의 마이크로 프런트엔드 모드는 바로 모놀리식 애플리케이션을 해결하는 좋은 방법입니다.

사례 NoneOS를 참고할 수 있습니다. 이는 ofa.js로 개발된 운영체제로, 규모가 매우 큽니다.

백엔드 개발자인데, 그냥 백엔드 템플릿 엔진으로 개발하고 싶어. ofa.js 써도 될까?

ofa.js의 SSR은 Node.js에 종속되지 않습니다. SCSR 규칙만 준수한다면, 어떤 백엔드 언어로든 템플릿 엔진 개발을 할 수 있습니다.

자세한 내용은 ofa.js SSR 을 참고하세요.

jQuery의 정신적 계승자

템플릿 구문을 지원하면서도 직접 노드를 조작할 수 있어 유연성과 효율성을 모두 잡았습니다

API 설계는 jQuery와 유사하지만 더 직관적입니다

      
// jQuery 코드
$("#target").html("some html code"); // html 설정
$("#target").text("set text"); // text 설정
var ele_text = $("#target").text(); // text 가져오기
var parents = $("#target").parents(); // 모든 부모 계층 배열 가져오기
var child = $("#target").children()[0]; // 첫 번째 자식 요소 가져오기
      
    
      
// ofa.js 코드
$("#target").html = "some html code";  // html 설정
$("#target").text = "set text"; // text 설정
var ele_text = $("#target").text; // text 가져오기
var parents = $("#target").parents; // 모든 부모 계층 배열 가져오기
var child = $("#target")[0]; // 첫 번째 자식 요소 가져오기
