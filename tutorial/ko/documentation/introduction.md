# 소개

## ofa.js란 무엇인가요?

ofa.js는 매우 사용하기 쉬운 Web 프론트엔드 프레임워크로, Web 애플리케이션 구축을 위해 설계되었으며, 개발 효율성을 높이고 개발 장벽을 낮추는 것을 목표로 합니다.

> 만약 HTML, CSS, JavaScript에 **이미 기본적으로 익숙하다면**, ofa.js는 매우 좋은 선택입니다.

## 왜 ofa.js를 개발했나요?

비록 React, Vue 등 프론트엔드 프레임워크가 이미 존재하지만, 이 프레임워크들은 모두 상대적으로 높은 진입 장벽을 가지고 있습니다. 개발자는 먼저 Node.js 설치, npm을 사용하여 패키지 설치, 스캐폴딩 구축 등 일련의 선행 지식을 학습해야 합니다.

이러한 초기 단계들은 실제 개발에 필수적이지 않을 뿐만 아니라, 프로젝트 시작 단계의 "블랙박스"가 되어 애플리케이션이 처음부터 불필요한 복잡성을 짊어지게 만듭니다. ofa.js의 탄생은 바로 이러한 딜레마를 타파하기 위한 것입니다.

## ofa.js의 존재 목적

### 프론트엔드 개발 프로세스 간소화

ofa.js는 전통적인 프론트엔드 개발의 번거로운 과정을 단순화합니다. 개발자는 Node.js, npm, 스캐폴딩과 같은 도구를 배우지 않고도 프론트엔드 애플리케이션을 빠르게 구축할 수 있습니다. 공식적으로 제공되는 빌드 프로그램(웹 버전)을 브라우저에서 열고, 로컬 디렉토리를 선택하기만 하면 개발을 시작할 수 있습니다.

### 개발 진입 장벽 낮추기

많은 사람들은 자신만의 애플리케이션을 개발하거나 개발자가 되기를 원하지만, 전통적인 Web(안드로이드, iOS 등) 개발 루트에 직면하면 개발 환경 설치, 스캐폴딩 구성 등 수많은 선행 지식을 익혀야 하기에 많은 초보자들이 어려움을 느끼고 포기한다.

ofa.js는 모든 것을 간단하게 만드는 데 전념합니다. Chrome 브라우저 하나만 있으면 웹페이지를 열어 애플리케이션을 구축할 수 있으며, 로컬 디렉터리를 바로 선택하고 열어 개발 전 준비 작업을 생략합니다. 모든 계산, 데이터 및 저장 작업은 사용자의 로컬에서 수행되며 클라우드 서비스에 의존하지 않습니다.

### 마이크로 프론트엔드의 궁극적인 솔루션이 되다

ofa.js는 웹 프론트엔드 개발을 마이크로서비스처럼 세분화하여 여러 독립적인 모듈로 나눌 수 있게 합니다. 각 모듈은 독립적으로 개발하고 독립적으로 배포할 수 있습니다. 전통적인 웹 프론트엔드의 경계가 깨진 후, 프론트엔드 기술은 점차 서버 기술의 경계를 넘어설 것입니다.

> 마이크로서비스(Microservices)는 대규모이며 복잡한 애플리케이션을 복수의 세분화된, 독립적으로 배포되고 실행되는 소규모 서비스로 분할하는 소프트웨어 아키텍처 스타일입니다.
