# 生産とデプロイ

ofa.jsを使って開発したプロジェクトは、静的サーバーに直接デプロイすれば使用できます。

## 開発環境

公式の [ofa Studio](https://core.noneos.com/?redirect=studio) を使用して開発できます。ワンクリックでプロジェクトの作成とプレビューが可能です。

静的なサーバーを自分で構築することもできます：

* Nginx や Apache などの静的サーバーソフトウェアを使用する
* Node.js の [http-server](https://www.npmjs.com/package/http-server) モジュールを使用する
* エディタの静的サーバープラグインを直接使用してプレビューする

## 本番環境

### プロジェクトをエクスポート

もし [ofa Studio](https://core.noneos.com/?redirect=studio) を使ってプロジェクトを構築している場合は、ツールに付属しているエクスポート機能を直接使用してください。

手動で構築したプロジェクトの場合は、プロジェクトフォルダをそのまま静的サーバーにデプロイすればよく、開発環境のモードと一貫性を保ちます。

### 圧縮難読化

本番環境では通常、ファイルサイズを削減し読み込み速度を向上させるために、圧縮・難読化ツールを使用する必要があります。[Terser CLI](https://terser.org/docs/cli-usage/) を使用して圧縮・難読化を行うことができます。

コマンドラインツールを使用したくない場合は、[ofa build](https://builder.ofajs.com/) を使用してオンラインでファイルの圧縮と難読化を行うことができます。このツールは現在ベータ版で、後日ofa Studioに統合される予定です。
