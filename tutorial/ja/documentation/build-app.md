# 本番とデプロイ

ofa.js で開発されたプロジェクトは、静的サーバーに直接デプロイして使用できます。

## 開発環境

あなたは公式の [ofa Studio](https://core.noneos.com/?redirect=studio) を使用して開発することができ、ワンクリックでのプロジェクト作成とプレビューを提供します。

、静的サーバーを自作することもできます：

* Nginx や Apache などの静的サーバーソフトウェアを使用する
* Node.js の [http-server](https://www.npmjs.com/package/http-server) モジュールを使用する
* 直接エディターの静的サーバープラグインを使用してプレビューする

## 本番環境

### プロジェクトのエクスポート

もし [ofa Studio](https://core.noneos.com/?redirect=studio) を使用してプロジェクトを構築している場合は、ツールに組み込まれているエクスポート機能を直接使用できます。

手動でビルドしたプロジェクトの場合、プロジェクトフォルダを静的サーバーに直接デプロイするだけでよく、開発環境のモードと同じように保てます。

### 圧縮と難読化

本番環境では通常、ファイルサイズを削減し読み込み速度を向上させるために圧縮・難読化ツールを使用します。[Terser CLI](https://terser.org/docs/cli-usage/) を使用して圧縮・難読化を行うことができます。

もしコマンドラインツールを使用したくない場合は、[ofa build](https://builder.ofajs.com/) をオンラインで使用してファイルの圧縮と難読化を行うことができます。このツールは現在ベータ版であり、後期には ofa Studio に統合される予定です。
