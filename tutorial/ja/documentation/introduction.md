# 紹介

## ofa.jsとは何ですか？

ofa.js は非常に使いやすい Web フロントエンドフレームワークで、Web アプリケーションを構築するために設計されており、開発効率を向上させ、開発のハードルを下げることを目的としています。

>HTML、CSS、そして JavaScript に **すでに基本的に慣れている** なら、ofa.js はとても良い選択です。

## なぜ私は ofa.js を開発するのか？

ReactやVueなどのフロントエンドフレームワークは既に存在しますが、どれも習得のハードルが高いです。開発者はまず、Node.jsのインストール、npmを使ったパッケージのインストール、スキャフォールディングの構築など、一連の前提知識を学ぶ必要があります。

これらの初期ステップは、実際の開発において不可欠なものではなく、むしろプロジェクトの立ち上げ段階における「ブラックボックス」となり、アプリケーションを最初から不必要な複雑さを背負わせています。ofa.js の誕生は、まさにこのジレンマを打破するためなのです。

## ofa.jsが存在する目的

### フロントエンド開発プロセスの簡素化

ofa.jsは、従来のフロントエンド開発における煩雑な手順を簡素化します。開発者はNode.js、npm、スキャフォールドなどのツールを学ぶことなく、フロントエンドアプリケーションを素早く構築できます。ブラウザで公式が提供するビルドプログラム（Web版）を開き、ローカルディレクトリを選択するだけで、開発を開始できます。

### 開発のハードルを下げる

多くの人々は自分自身のアプリケーションを開発したい、または開発者になりたいと考えていますが、従来のWeb（Android、iOSなどを含む）開発ルートに直面すると、開発環境のインストールやスキャフォールディングの設定など、多くの前提知識を学ぶ必要があるため、多くの初心者が困難を感じて諦めてしまいます。

ofa.jsは、すべてをシンプルにすることを目指しています。Chromeブラウザーひとつあれば、ウェブページを開くだけでアプリケーションを構築でき、ローカルディレクトリを直接選択して開くため、開発前の準備作業が不要です。すべての計算・データ・ストレージ処理はユーザーのローカル環境で実行され、クラウドサービスへの依存はありません。

### マイクロフロントエンドの究極のソリューションへ

ofa.js は、Webフロントエンド開発をマイクロサービスのように細分化し、各モジュールを独立して開発・デプロイ可能にします。従来のWebフロントエンドの境界が打破されると、フロントエンド技術は徐々にサーバー技術の境界を突破していくでしょう。

> マイクロサービス（Microservices）は、大規模で複雑なアプリケーションを、細かい粒度で独立してデプロイ・実行可能な複数の小さなサービスに分割するソフトウェアアーキテクチャスタイルです。
