# PATH



`PATH` 属性は通常、カスタムコンポーネントまたはページコンポーネントで使用され、そのコンポーネントの登録ファイルのアドレスを取得するために使われます。これは開発過程でコンポーネントの出所を理解するのに役立ち、特に他のリソースファイルを参照またはロードする必要がある場合に、`PATH` 属性を使ってファイルパスを構築できます。

以下は、カスタムコンポーネントで `PATH` プロパティを使用する方法を示す簡単な例です：

```javascript
const componentPath = $("#myCustomComponent").PATH;
```

この例では、`id` が "myCustomComponent" の `my-comp` 要素を選択し、`PATH` 属性を介してそのカスタムコンポーネントのファイルパスを取得しました。スクリプト部分で必要に応じて `componentPath` 変数を使用できます。例えば、それを使って他のリソースファイルのパスを構築したり、他の操作を行ったりできます。
