もう学ぶの疲れた、ドキュメント読みたくない、使える？

ofa.jsを先に学ぶ必要はなく、それに基づいて開発されたコンポーネントを直接再利用できます。

オープンソースコンポーネントの作者がコードを書いた後、他の人は例に従って数行コピーするだけで、瞬時に機能を自分のプロジェクトに取り込めます。

古いプロジェクトに一時的に機能を追加したい

既存のプロジェクトでscriptタグを通じてofa.jsを導入するだけで、すぐに使用できます。

導入後、様々なコンポーネントが手に入ります。

開発初心者で、Nodejs と npm に挫折した

ofa.jsは新人にとても優しく、Node.js、npmなどのツールを学ぶ必要はありません。

ブラウザ1つと任意の静的サーバーがあれば、開発を始められます。

Chromeブラウザを使っているなら、 OFA Studio というオンラインツールを開けば、静的サーバーを準備する必要はなく、直接始められます。

チュートリアル：最初のアプリを作成する
巨大なショッピングモールシステムを開発する、あなたたちは対応できる？

ofa.jsはマイクロフロントエンドのアーキテクチャパターンを採用しているため、大規模プロジェクトに非常に適しています。

ショッピングモールシステムを十分に細かい要件に分割し、コンポーネントごとに開発し、独立したフォルダに配置すれば、静的サーバーを通じて直接コンポーネントの効果をプレビューできます。

最後に一つずつ組み立てて巨大なシステムにします。ofa.jsのマイクロフロントエンドモードは、まさにモノリシックアプリケーションの特効薬です。

参考ケース： NoneOS、これはofa.jsで開発されたオペレーティングシステムで、規模が非常に大きいです。

バックエンド開発者です、バックエンドのテンプレートエンジンだけで開発したい、ofa.js は適していますか？

ofa.jsのSSRはNode.jsに縛られず、SCSRルールに従えば、任意のバックエンド言語でテンプレートエンジン開発ができます。

詳細は ofa.js SSR を参照してください。

jQueryの精神的な継承者

テンプレート構文をサポートしつつ、直接ノードを操作でき、柔軟性と効率性を両立

API設計はjQueryに似ていますが、より直感的です

      
// jQuery コード
$("#target").html("some html code"); // htmlを設定
$("#target").text("set text"); // テキストを設定
var ele_text = $("#target").text(); // テキストを取得
var parents = $("#target").parents(); // すべての親レイヤーの配列を取得
var child = $("#target").children()[0]; // 最初の子要素を取得
      
    
      
// ofa.js コード
$("#target").html = "some html code";  // htmlを設定
$("#target").text = "set text"; // テキストを設定
var ele_text = $("#target").text; // テキストを取得
var parents = $("#target").parents; // すべての親レイヤーの配列を取得
var child = $("#target")[0]; // 最初の子要素を取得
